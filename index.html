BAM — Custom Game Engine

C++ Lua Dear ImGui SDL2 Box2D LuaBridge GLM RapidJSON

BAM is a 2D game engine I built in C++ with a Lua scripting runtime. The main thing I added on top of the base engine is an integrated visual tile map editor using Dear ImGui.

The editor and the game are the same binary — passing --play runs the game, no args opens the editor. You paint a tile map, hit Save, and the engine rewrites the stage1 table directly in your Lua scripts. No separate map format, no import step.

The brush palette builds itself by parsing GameManager.lua at startup — it looks for tile_code checks and the Instantiate calls next to them, so adding a new tile type to your Lua automatically shows up in the editor without touching any config file.
